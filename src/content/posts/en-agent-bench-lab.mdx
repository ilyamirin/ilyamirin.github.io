I built `agent-bench-lab` to compare coding agents on the same real Python task, in the same environment, with the same acceptance flow.

The core question was not “which agent writes the best code.”

The real question was whether an agent could close the full loop: understand the task, edit the right layers, respect project constraints, and survive verification.

Most systems can produce plausible code. Far fewer can finish engineering work all the way through migrations, tests, visible-surface wiring, and browser-level acceptance.

That is why the benchmark signal I care about is operational closure, not raw code generation.
