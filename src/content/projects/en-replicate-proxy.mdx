This project treats model access as a systems problem.

Instead of binding an application to one provider, `replicate_proxy` creates an OpenAI-compatible access layer that can route, normalize, and preserve application behavior.

What it proves about me:

- I can build useful compatibility layers
- I understand model access as infrastructure, not just integration glue
- I care about stable interfaces under changing providers
